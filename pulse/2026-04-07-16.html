Harness Engineering: OpenAI's Frontier Team Ships 1M LOC Without Writing Code

Latent Space

Ryan Lopopolo of OpenAI's Frontier team reveals how they built an internal product over five months with zero lines of human-written code—a million lines in the repo, thousands of PRs across multiple Codex generations. The constraint was deliberate: Ryan refused to write code himself, forcing the team to build systems, observability, and context that let agents review, fix, and merge work autonomously.

The key shift: humans became the bottleneck, not the agents. Most human review is now post-merge. They shifted from reviewing code directly to building systems that encode engineering taste into context the agent can use—skills, docs, tests, markdown trackers, and quality scores. The result is Symphony, an Elixir-based orchestration layer for coordinating large numbers of coding agents across repos.

"I do think that if you aren't using more than a billion tokens a day, it's borderline negligent... roughly $2-3k/day in token spend based on market rates."

GitHub Traffic Surges 14x as AI Agents Flood the Platform

The Information

GitHub COO Kyle Daigle reveals the staggering scale of AI-driven code generation: commits have surged from 1 billion annually to 14 billion projected this year—a 14x increase. Pull requests from AI agents jumped from 4 million in September to over 17 million in March. Claude Code's public GitHub commits increased 25x in six months.

The surge coincides with OpenAI's Codex release in February and Anthropic's improved Claude models. Open source tools like OpenClaw that harness multiple agents are accelerating adoption. GitHub is scrambling to add servers and rebuild infrastructure—the platform has seen more outages as it migrates to Azure while traffic explodes.

"Since January, every month, every week almost now has some new peak stat for the highest usage rate ever."
