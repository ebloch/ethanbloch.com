Notion Engineers Are Abandoning Cursor for Claude Code and Codex

The Information

Hundreds of engineers at Notion are switching from Cursor to Anthropic's Claude Code and OpenAI's Codex. The shift represents a broader trend: engineers moving from IDE-integrated tools to fully agentic coding assistants that complete entire tasks autonomously.

The usage pattern is fascinating. Junior engineers prefer Claude Code for simpler tasks — it's good at anticipating what you want without needing precise instructions. Senior engineers run Codex overnight for complex, long-running tasks, sometimes letting agents work continuously for eight hours while they sleep.

"Our software engineers 100% use coding agents now. Many of them haven't generated a line of code in a while, but they're super productive."

For Hiro: This is the clearest signal yet that AI coding has crossed from 'productivity boost' to 'fundamental workflow change.' For Hiro, the parallel is obvious: if senior engineers trust agents to run unsupervised for 8 hours, why wouldn't investors eventually trust AI advisors with similar autonomy? The Claude Code revenue numbers also validate that people will pay real money for capable agents.

a16z to Software CEOs: Pick a Path or Die in the Middle

a16z

A stark memo from a16z to software founders: the comfortable middle is over. Public markets have already repriced software because terminal value isn't what it used to be. The firm sees only two viable paths forward.

Path One: Accelerate revenue growth by 10+ points through genuinely new AI-native products in 12-18 months. Not chatbot add-ons — new products that move the needle. Put 50% of R&D on net-new AI products using four-person pods. Collapse design, product, and engineering into single units. Cap headcount, not compute. Path Two: Rebuild for 40%+ true operating margins (including stock comp as a real expense) within 12-24 months. This isn't 10-20% RIFs. It's flattening management, killing committees, standardizing implementation, raising prices where you own the workflow, and letting low-value customers churn.

"Founders: you need to pick a path, and you need to decide quickly who on your team you want on the bus with you. You have the opportunity for a new founding moment for your company."

For Hiro: This is the strategic framework for thinking about Hiro's category. Traditional fintech software (Yodlee, Plaid, legacy WealthTech) is exactly the 'comfortable middle' that's being repriced. Hiro's bet is Path One — AI-native products that aren't just features bolted onto existing SKUs. The 'four-person pod' model and 'cap headcount, not compute' mantras are worth internalizing.
